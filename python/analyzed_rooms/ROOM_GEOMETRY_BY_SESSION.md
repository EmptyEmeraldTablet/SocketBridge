### top_left / bottom_right 坐标含义

`room_info` 中的 `top_left` 和 `bottom_right` 字段定义的是**墙壁内边界**（可移动区域的边界），而非墙壁外边界：

| 字段 | 含义 | 包含 |
|------|------|------|
| `top_left` | 墙壁内边界左上角坐标 | 可移动区域左上角 |
| `bottom_right` | 墙壁内边界右下角坐标 | 可移动区域右下角 |

**坐标与区域关系图示:**

```
┌─────────────────────────────────────────────────┐
│                   墙壁外边界                       │
│           (grid × 40 像素范围)                     │
│  ┌─────────────────────────────────────────┐     │
│  │         top_left (60, 140)               │     │
│  │  ┌───────────────────────────────────┐   │     │
│  │  │                                   │   │     │
│  │  │         可移动区域                  │   │     │  ← bottom_right - top_left
│  │  │      ((grid-2)×40 像素)            │   │     │    = 可移动区域像素尺寸
│  │  │                                   │   │     │
│  │  └───────────────────────────────────┘   │     │
│  │        bottom_right (1100, 700)          │     │
│  └─────────────────────────────────────────┘     │
│                   墙壁外边界                       │
└─────────────────────────────────────────────────┘

墙壁厚度: 上下左右各1格 (40像素)
可移动区域: (grid_width - 2) × (grid_height - 2) × 40 × 40
```

**计算公式验证:**

```
像素宽度  = bottom_right[0] - top_left[0]
          = (grid_width - 2) × 40

像素高度  = bottom_right[1] - top_left[1]
          = (grid_height - 2) × 40
```

**示例 (Room 71, shape=11, grid=28×16):**

| 参数 | 值 |
|------|------|
| grid_width × grid_height | 28×16 |
| top_left | (60.0, 140.0) |
| bottom_right | (1100.0, 700.0) |
| 像素尺寸 | 1040×560 |
| 期望值 | (28-2)×40 × (16-2)×40 = 1040×560 ✅ |

## L形房间缺角区域分析

### 当前 room_info 数据能否完全确定缺角区域？

**答案：✅ 可以完全确定**

四种 L 形房间的缺角都是通过可移动区域的**中心点**裁剪掉 1/4 矩形，因此可以直接计算缺角边界。

### 核心公式

```
center_x = (top_left[0] + bottom_right[0]) / 2
center_y = (top_left[1] + bottom_right[1]) / 2
```

### 缺角边界计算

| Shape Code | 缺角位置 | 缺角边界 | 尺寸 |
|------------|----------|----------|------|
| 9 | 左上 | `(top_left) - (center_x, center_y)` | 520×280 |
| 10 | 右上 | `(center_x, top_left[1]) - (bottom_right[0], center_y)` | 520×280 |
| 11 | 左下 | `(top_left[0], center_y) - (center_x, bottom_right[1])` | 520×280 |
| 12 | 右下 | `(center_x, center_y) - (bottom_right)` | 520×280 |

### 数据验证

| 房间 | Shape | 缺角类型 | 可移动区域 | 中心点 | 缺角边界 |

## 算法说明

### v2.0 形状检测算法

基于参考算法实现，从边界采样点还原图形：

1. **房间大小验证**: 检查房间是否足够大且 shape code 允许 L 形
2. **墙壁推断**: 对每个采样点计算到各墙壁的距离，推断其所在的墙壁
3. **覆盖率分析**: 计算各面墙的采样覆盖范围比例
4. **网格密度分析**: 使用 4×4 网格统计采样点密度，检测稀疏区域
5. **综合判断**: 融合多信号判断房间形状和缺失角落

### L 形检测特征

- 某面墙的覆盖率明显偏低（<60%）
- 采样点分布偏向一侧（两端缺失）
- 网格密度分析发现稀疏区域
